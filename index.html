Declarative

Imperative: “How to do something”

Declarative: “What to do”

Benefits

  • Reduce side effects
  • Minimize mutability
  • More readable code
  • Less Bugs
  • Context-independent

Imperative

								
									function double (arr) {
									let results = []

									for (let i = 0; i < arr.length; i++){
										results.push(arr[i] * 2)
									}

									return results
									}

Declarative

								
									function double (arr) {
										return arr.map((item) => item * 2)
									}

React Ecosystem

Entirely possible to build a React app with no other dependencies, but most apps use a variety of additional libraries for specific capabilities.

  • Good news: can pick exactly the libraries you need for your use cases.
  • Bad news: need to pick exactly the libraries you need for your use cases.

Common libraries

  • AJAX requests: Axios, SuperAgent, Fetch
  • Routing: React-Router
  • State management: Redux, MobX
  • Data/utilities: Immutable.js, Lodash

Build tools

  • Babel: compiles ES6/JSX to widely-compatible ES5
  • Webpack: reads multiple JS module formats, loads non-JS formats such as CSS and images, runs Babel, and generates optimized output bundles for deployment

But how do I manage state once the application gets more complex?

What is state management?

“The state of a program or application is defined as its condition regarding stored inputs. The term “state” here is used similarly to how it is used in science – whereas the state of an object, for instance a gas, liquid, or solid, shows its current physical makeup, the state of an application shows its current values or contents.” - Technopedia

Examples of things you would store in state

  • User input (form data, button clicks, etc.)
  • Server responses (API response)
  • Locally generated data (data you've created but not persisted)

What is Redux?

"A predictable state container for JavaScript applications"

Three Principles of Redux

  1. The whole state of your app is stored in an object tree inside a single store
  2. The only way to change the state tree is to emit an action, an action describing what happened. (state is immutable -- it cannot be modified directly)
  3. To specify how actions transform the state tree, you write pure reducers. (“pure” === no side effects)

Redux Terminology

Store

  • Redux has a single store that holds application information
  • Role is to receive actions and use reducers to calculate new state and save it
  • Notifies all registered listeners when a state change occurs

Actions

Plain Javascript objects that contain information about how the store should modify state


						{
							type: 'ADD_TODO',
							text: 'Get a haircut'
						}

Action Creators

Functions that create actions


						function addTodo(text) {
							return {
								type: 'ADD_TODO',
								text
							}
						}

Reducer

Specify how state changes in response to actions


						function todoApp(state = initialState, action) {
						  switch (action.type) {
						    case ADD_TODO:
						      return Object.assign({}, state, {
						        newestTodo: action.text
						      })
						    default:
						      return state
						  }
						}
